
# 料理レシピDSL可視化ツール (Hamko Recipe DSL Visualizer)

このプロジェクトは、テキストベースの「料理レシピ言語 (DSL)」を解析し、フローチャートとして可視化するツールです。

## 機能
*   **レシピDSLエディタ**: 直感的な構文でレシピを記述できます。
*   **フローチャート可視化**: 記述したレシピをリアルタイムでフローチャートに変換します。
*   **材料・手順リスト**: DSLから自動的に材料一覧と調理手順を抽出して表示します。
*   **ツールチップ**: 材料にカーソルを合わせると、登録されている画像を表示します。
*   **レシピ検索**: レシピ名での検索、評価（★）によるフィルタリングが可能です。
*   **AI生成機能**: 自然言語のレシピテキストから、OpenAI APIを使用してDSLを自動生成します。

## レシピDSL言語の名前候補
この言語の名称案です：
1. **Hamko Recipe** (Req)
2. **CookFlow DSL**
3. **RecipeGraph**
4. **CulinaryCode**
5. **GourmetScript**
6. **KitchenNodes**
7. **DishDiagram**
8. **ChefSyntax**
9. **FlavorFlow**
10. **EatAlgorithm**

## DSLの書き方
基本文法は以下の通りです：

*   **材料**: `豚バラ肉300` `塩少々`
*   **操作**: `@操作名[引数]` (例: `@SauteL5` = 弱火で5分炒める)
*   **グループ化**: `(材料A 材料B)`
*   **結合**: `++` (例: `(具材) ++ (調味料)`)
*   **遷移**: `->` (暗黙的にも遷移します)
*   **コメント**: `{コメント}` (例: `@Wait30{常温で}`)

### 例
```
(豚バラ300@Cut)@SauteL5 ++ (玉ねぎ1個@Slice)@Saute 
-> 調味料(醤油30 酒30 砂糖10)@Mix 
-> @BoilL20
```

## GitHub Pagesについて
このアプリケーションはGithub Pagesのような静的ホスティングで動作しますが、以下の点に注意してください：
*   **AI生成機能**: クライアントサイドからOpenAI APIを直接呼び出します。APIキーはブラウザのLocalStorageに保存されますが、公開されたページ上で不特定多数に利用させる場合は、APIキーの管理（流出リスク）に注意が必要です。個人利用を想定しています。
*   **データ保存**: 作成したレシピの保存機能は現在LocalStorage等には対応していませんが、DSLテキストをコピーすることで保存可能です。
